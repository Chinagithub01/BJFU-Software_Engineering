**一、项目背景**

NekoCafé 是一家面向年轻用户的连锁猫咪主题餐厅集团，目前在全国 12
个城市开设 38
家门店。随着业务扩张，原有的门店独立排号系统已无法满足跨门店统一会员、智能推荐、宠物档案管理等新需求。

集团 CTO 委托学生团队设计并实现新一代「NekoCafé
智慧餐饮预约平台」，本题与本学期 4
个软件工程实验同案例，可复用实验阶段产出的需求基线、架构设计、CI/CD
流水线与测试用例。

**二、业务目标**

> **1.** 用户可通过移动端 / 小程序 / Web 完成
> 门店浏览、桌位预约、菜品点单、会员管理；
>
> **2.** 系统须支持 AI
> 智能推荐（根据用户偏好与同行猫咪性格匹配桌位与菜品）；
>
> **3.** 提供 店员端 用于桌位调度、订单处理与猫咪健康档案管理；
>
> **4.** 支持 运营端 数据看板（坪效、翻台率、会员复购率）；
>
> **5.** 高峰期支持 1000 并发预约，核心接口 P95 ≤ 350 ms。

**三、用户角色**

  **角色**                **主要诉求**                   **权限范围**
  ----------------------- ------------------------------ --------------------------------
  顾客 (Customer)         一键预约、推荐精准、价格透明   浏览 / 预约 / 点单 / 评价
  店员 (Staff)            操作高效、调度合理、异常告警   订单处理 / 桌位调度 / 猫咪打卡
  店长 (Manager)          门店经营数据、人员排班         门店级数据 / 人员管理
  总部运营                跨门店分析、活动投放           全局数据 / 活动配置
  猫咪管家 (Cat Keeper)   猫咪健康档案、互动记录         猫咪档案 / 健康打卡

**四、功能模块**

**4.1 必做功能（核心交付，≥ 5 个模块）**

> **•** M1 用户与会员：注册 / 登录 / 实名认证 / 会员等级 / 积分；
>
> **•** M2 门店与桌位：门店列表 / 桌位库存 / 实时排队；
>
> **•** M3 预约与点单：可视化选桌 / 菜品下单 / 在线支付（支付沙箱）；
>
> **•** M4 订单与履约：订单状态机 / 取消改约 / 退款；
>
> **•** M5 店员后台：接单 / 调度 / 异常告警；
>
> **•** M6 数据看板：坪效 / 翻台率 / 会员复购率（折线 / 柱状 / 饼图）。

**4.2 选做功能（加分项，每实现 1 个 +2 分）**

> **•** AI 推荐：基于用户偏好和\"同行猫咪性格\"匹配桌位与菜品；
>
> **•** 猫咪健康档案：体重 / 疫苗 / 互动记录 + 趋势图；
>
> **•** 小程序端 + 微信支付沙箱接入；
>
> **•** 基于位置的\"距我最近\"门店检索（LBS）；
>
> **•** 运营活动引擎：满减 / 折扣 / 优惠券叠加规则。

**五、技术约束**

  **类别**   **约束 / 推荐**
  ---------- --------------------------------------------------
  后端框架   Spring Boot 3 / FastAPI / NestJS 任选
  前端框架   Vue 3 / React 18 + 微信小程序（选做）
  数据库     PostgreSQL 16 + Redis 7
  消息队列   RabbitMQ / Kafka（用于订单异步处理）
  鉴权       JWT + 短信验证码（沙箱）
  容器化     Docker + docker-compose 一键起栈
  CI/CD      GitHub Actions（lint / test / build / 镜像推送）
  可观测性   Prometheus + Grafana（选做）

**六、验收标准**

> **1.** 完成 6 个必做模块，UI 美观、操作流畅；
>
> **2.** 系统须能在评委本地通过 docker compose up 一键启动；
>
> **3.** 至少完成 1 项选做功能；
>
> **4.** 单元测试覆盖率 ≥ 60%（核心域 ≥ 80%）；
>
> **5.** 提交 k6 / JMeter 压测脚本与结果；
>
> **6.** 提交完整的 12 份产出文件 + 5 份管理表单。

**七、典型风险与建议**

  **风险**         **影响**   **建议**
  ---------------- ---------- ---------------------------------------------
  AI 推荐难度      高         可降级为基于偏好标签的规则推荐
  跨设备调试       中         小程序端可降级为 Web H5 演示
  性能压测环境     中         可在本地容器内压测，关注 P95 趋势而非绝对值
  进度估算偏乐观   高         坚持每日早会 + 每周周报，及时调整范围
