**一、项目背景**

高等教育越来越强调过程性评价。让学生互相评阅作业，既能减轻教师负担，又能促进同伴学习。然而，传统的纸质互评效率低、匿名性差、统计困难。学院希望开发一个面向多门课程的互评平台，支持作业发布、提交、随机派评、匿名互评、申诉与复评。

**二、业务目标**

> **1.** 教师可在平台发布作业 + 评分量规（Rubric）；
>
> **2.** 学生提交作业后，系统按算法分配 N 篇同伴作业进行互评；
>
> **3.** 支持匿名互评 + 评论 + Rubric 打分；
>
> **4.** 学生对低分有申诉权，教师复评仲裁；
>
> **5.** 提供统计分析与诚信检测（如互评分差异检测）。

**三、用户角色**

  **角色**   **主要诉求**                   **权限范围**
  ---------- ------------------------------ ---------------------
  教师       发布作业、设定规则、复评仲裁   课程级管理员
  学生       提交、互评、查看反馈、申诉     本人作业 + 派评作业
  助教       协助派评 / 处理常见问题        介于教师与学生之间
  管理员     维护平台与课程目录             系统级

**四、功能模块**

**4.1 必做功能（核心交付，≥ 5 个模块）**

> **•** M1 课程与作业：课程创建 / 学生导入 / 作业发布 / Rubric 配置；
>
> **•** M2 提交与查重：作业上传（PDF/Doc/Code）+ 代码相似度提示；
>
> **•** M3 派评算法：随机 / 匿名 / 防止互评（A 评 B & B 评 A）；
>
> **•** M4 互评界面：分项打分 + 评论 + 总分自动汇总；
>
> **•** M5 反馈与申诉：学生收到反馈、低分申诉、教师复评；
>
> **•** M6 数据统计：分数分布、互评一致性、异常检测。

**4.2 选做功能（加分项，每实现 1 个 +2 分）**

> **•** AI 辅助评分：对比同学评分给出建议；
>
> **•** 雷达图展示 Rubric 各维度得分；
>
> **•** 导出 Excel 报表给教务系统；
>
> **•** 匿名讨论区：互评后限时讨论。

**五、技术约束**

  **类别**   **约束 / 推荐**
  ---------- ----------------------------
  前端       React 18 / Vue 3
  后端       NestJS / Django 5
  数据库     PostgreSQL 16
  文件存储   MinIO
  鉴权       JWT + 校园 SSO（CAS 模拟）
  查重       集成 simhash 简单算法

**六、验收标准**

> **1.** 完成 6 个必做模块；
>
> **2.** 匿名性须可被代码与数据库审计验证；
>
> **3.** 派评算法须公布伪代码，并可证明无系统性偏差；
>
> **4.** 至少 1 门\"演示课程\"包含 ≥ 30 名虚拟学生数据；
>
> **5.** 提交完整的 12 份产出文件 + 5 份管理表单。

**七、典型风险与建议**

  **风险**       **影响**   **建议**
  -------------- ---------- ---------------------------------------
  匿名性破坏     高         所有评论审计日志单独脱敏存储
  派评算法争议   中         提前公示算法 + 可重放种子
  抄袭检测漏洞   中         说明 simhash 仅作辅助，最终由教师裁决
  UI 信息密度    低         使用 Tailwind / Ant Design 标准化组件
